# RAG 课程助手 — 设计文档

## 数据流

```
course-faq.md → parse_faq() → retrieve() → Prompt 拼接 → LLM → 输出
     ↓              ↓              ↓              ↓
  10条FAQ      切片为chunk      关键词匹配      检索结果+问题    生成回答+来源
```

## 检索策略

### 分词方案

中文按 2-gram + 英文按空格小写：
- 中文字符提取正则：`[\u4e00-\u9fff]`
- 英文单词提取正则：`[a-z0-9]+`
- 中文 2-gram：相邻两字组合，捕捉词级语义
- 过滤泛化疑问词：如"什么"、"如何"、"这个"，避免资料外问题因为常见词误召回
- 对"Day1 要交什么"这类课程交付问题做小范围同义扩展：`提交`、`交付`、`证据`、`文件`

### 评分公式

```
score = (query token 命中数 / query token 总数) + 标题加权 + 课程交付问题加权
```

- 标题加权：query token 出现在标题中，额外 +0.3
- 课程交付问题加权：当用户问"Day1 要交什么"时，优先匹配"6 类证据文件"和"最终提交包"相关条目
- 阈值过滤：score < 0.25 的结果丢弃

### 返回结果

按 score 降序排列，返回 top_k=3 个 chunk

## 拒答逻辑

三道防线：

1. **空输入**：query 为空或纯空格 → 提示"请输入您的问题"
2. **无结果**：retrieve() 返回空列表 → 拒答
3. **低分**：最高分 < 0.25 → 拒答
4. **超范围问法**：包含"没有教"、"除了...还有..."等要求资料外扩展的问法 → 拒答
5. **拒答后处理**：如果模型回答中包含"资料中没有找到依据"，程序不再附加来源编号

拒答回答："资料中没有找到依据。本助手仅能回答 Day1 AI Native 训练营相关问题。"

## Prompt 模板

### System 指令

```
你是 Day1 AI Native 训练营的课程助手。

规则（必须严格遵守）：
1. 只基于下方【课程资料】回答问题，不要使用任何外部知识
2. 如果资料中没有相关内容，直接回答"资料中没有找到依据"
3. 不要在回答中输出来源编号，系统会自动添加
4. 用中文回答，语言简洁准确
```

### User 消息

```
【课程资料】
---
【faq-XX】标题
内容
---

【用户问题】
问题内容

请基于课程资料回答，不要输出来源编号，系统会自动添加来源。
```

## 设计权衡与失败模式

本项目没有使用 embedding 或向量数据库，是基于场景规模的选择：知识库只有 10 条 FAQ，引入向量模型会增加依赖、配置和不可复现风险，而课程评分重点是工程闭环，不是检索算法复杂度。

关键词检索的主要风险是误召回。早期实现把中文单字也纳入 token，导致"奖学金政策""今天中午吃什么"这类资料外问题，因为命中了"什么""是"等泛词而被错误匹配到 FAQ。修正方案是：中文只使用 2-gram，并过滤泛化疑问词；同时提高检索阈值，减少弱相关结果进入 Prompt。

拒答逻辑不只依赖 Prompt。系统采用两层防线：检索前过滤空输入、低相关问题和明显超范围问法；模型返回后如果包含"资料中没有找到依据"，程序不再附加来源，避免出现"拒答但仍显示 [faq-XX]"的矛盾输出。
