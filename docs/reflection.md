# 复盘反思

## 认知变化

### 1. RAG 的本质

**之前理解**：RAG 是一个复杂的 AI 系统，需要向量数据库、embedding 模型等高级组件。

**现在理解**：RAG 的核心是"检索 + 增强 + 生成"三步：
- 检索：从知识库中找到相关文档
- 增强：将文档和问题拼接成 Prompt
- 生成：LLM 基于 Prompt 生成回答

**关键认知**：RAG 不是让 LLM 更聪明，而是让 LLM 有据可依。

### 2. 检索策略

**之前理解**：必须使用向量检索才能找到相关文档。

**现在理解**：关键词匹配在小规模知识库（10 条 FAQ）中完全够用：
- 中文 2-gram 能捕捉词级语义
- 标题加权能提高主题匹配度
- 阈值过滤能避免误匹配

**关键认知**：技术选型要匹配场景规模，不要过度设计。

### 3. 拒答逻辑

**之前理解**：拒答是 LLM 的事，Prompt 里写"不要编造"就行。

**现在理解**：拒答需要三道防线：
1. 空输入 → 提示用户
2. 无检索结果 → 直接拒答
3. 低分结果 → 直接拒答

**关键认知**：不能完全依赖 LLM 遵循指令，需要程序逻辑兜底。

### 4. 来源引用

**之前理解**：来源引用是 LLM 的事，Prompt 里要求输出就行。

**现在理解**：来源引用需要程序逻辑：
- 从检索结果中提取 [faq-XX]
- 从模型回答中提取来源
- 避免重复显示

**关键认知**：LLM 可能不遵循指令，需要程序逻辑保证。

## 遇到的困难

### 1. 接口契约不匹配

**问题**：test_basic.py 要求 retrieve() 返回 list，answer() 返回 dict，但初始实现抛出 NotImplementedError。

**解决**：改为返回空值，符合接口契约。

**教训**：先看测试要求，再写实现。

### 2. 来源重复显示

**问题**：模型回答中输出来源，main.py 又打印一次来源。

**解决**：从模型回答中提取来源，删除回答中的来源部分。

**教训**：LLM 输出不可控，需要程序逻辑处理。

### 3. Windows 终端编码问题

**问题**：Python 输出中文乱码。

**解决**：设置环境变量 `PYTHONIOENCODING=utf-8`。

**教训**：跨平台开发要注意编码问题。

### 4. API 配置

**问题**：Mock 服务和 MiMo API 切换不方便。

**解决**：使用环境变量配置 API 地址和密钥。

**教训**：配置应该外部化，不要硬编码。

## 改进方向

### 1. 检索优化

当前使用简单的关键词匹配，可以改进：
- 使用 TF-IDF 算法提高权重
- 使用同义词扩展提高召回率
- 使用语义相似度（需要 embedding 模型）

### 2. 拒答优化

当前使用阈值过滤，可以改进：
- 使用 LLM 二次确认是否拒答
- 使用置信度分数判断
- 使用多轮对话澄清问题

### 3. 来源引用优化

当前从模型回答中提取来源，可以改进：
- 使用结构化输出（JSON 格式）
- 使用后处理验证来源准确性
- 使用 Prompt 工程减少来源错误

### 4. 用户体验优化

当前是 CLI 工具，可以改进：
- 添加交互式问答模式
- 添加历史记录功能
- 添加导出功能
